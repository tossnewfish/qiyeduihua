# 企业知识库问答 Agent 设计说明

## 系统架构
系统由 FastAPI 后端、模型编排层、轻量级本地向量索引和简单 Web 页面组成。后端接收用户请求后，会读取最近的对话历史，按需调用本地检索工具，并最终返回带知识依据的回答。

## 工具策略
当前助手暴露两个工具：`search_kb` 用于语义检索知识库内容，`get_chunk_by_id` 用于按片段编号获取更完整的证据。对于依赖企业文档的问题，模型应优先调用 `search_kb`；只有在需要查看某个片段的更详细上下文时，才继续调用 `get_chunk_by_id`。

## 多轮上下文
系统通过 `session_id` 在内存中维护最近若干轮用户与助手消息。较早的历史对话会被截断，以控制上下文长度并降低推理开销。

## 评估思路
离线评估可以重点关注：检索命中质量、答案准确性、引用正确性，以及在知识库证据不足时是否能够稳定拒答或说明不确定性。
